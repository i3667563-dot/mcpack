# mcpack

Генератор датапаков для Minecraft с использованием ИИ (Ollama).

## Возможности

- **Два формата**: старый (≤1.20, по умолчанию 1.16.5) и новый (≥1.21, по умолчанию 1.21.1)
- **Генерация кода через ИИ**: использует Ollama для создания профессионального кода датапака
- **Встроенные лучшие практики**:
  - Fake players для scoreboard (`#counter`, `__name__`)
  - Правильная инициализация scoreboard с критерием `dummy`
  - Корректные теги функций (`minecraft:load`, `minecraft:tick`)
  - Namespaced ID и чистая структура папок
  - Правильное использование селекторов (`@p`, `@a`, `@e`, `@s` — никогда не `s` для датапаков)
- **Именование папок**: автоматически использует `functions` (старый) или `function` (новый) в зависимости от формата

## Установка

```bash
go build -o mcpack .
```

## Использование

```bash
mcpack [flags]
```

### Флаги

| Флаг | Короткий | Описание |
|------|----------|----------|
| `--old` | | Старый формат датапака (≤1.20, по умолчанию 1.16.5) |
| `--new` | | Новый формат датапака (≥1.21, по умолчанию 1.21.1) |
| `--prompt` | `-p` | Описание датапака для генерации (обязательно) |
| `--model` | `-m` | Модель Ollama (по умолчанию: `llama3.2`) |
| `--dir` | `-d` | Выходная директория (по умолчанию: текущая) |
| `--ollama` | | URL API Ollama (по умолчанию: `http://localhost:11434`) |
| `--verbose` | `-v` | Включить подробный вывод |
| `--dry-run` | | Сгенерировать структуру, но не записывать файлы |
| `--help` | `-h` | Показать справку |

**Примечание:** Короткие флаги `-o` и `-n` недоступны из-за конфликта с `--old`/`--ollama` и `--new`.

### Примеры

```bash
# Базовое использование (старый формат по умолчанию)
mcpack -p "создать систему scoreboard для отслеживания смертей игроков"

# Новый формат (1.21+)
mcpack --new -p "система кастомных зачарований" -m llama3.2

# Указать выходную директорию
mcpack --old -d ./mydatapack -p "команды телепортации"

# Подробный режим с dry-run
mcpack -n -p "система волн на арене мобов" --verbose

# Кастомная модель Ollama
mcpack --new -m mistral -p "помощники для автоматизации фермы"
```

## Форматы паков

| Формат | Версия Minecraft | Pack Format | Папка функций |
|--------|------------------|-------------|---------------|
| Старый | 1.16.5 (по умолчанию) | 6 | `functions` |
| Новый | 1.21.1 (по умолчанию) | 41 | `function` |

## Лучшие практики

ИИ инструктирован следовать этим лучшим практикам:

### Scoreboard
```mcfunction
# НЕПРАВИЛЬНО - не используйте случайных игроков
scoreboard objectives add deaths deathCount

# ПРАВИЛЬНО - используйте dummy с fake player
scoreboard objectives add deaths dummy "Смерти игроков"
scoreboard players set #deaths_counter deaths 0
```

### Теги функций
```json
// data/namespace/tags/function/load.json
{
  "values": ["namespace:load"]
}

// data/namespace/tags/function/tick.json
{
  "values": ["namespace:tick"]
}
```

### Селекторы
- Используйте `@p`, `@a`, `@e`, `@s` — **никогда** не надейтесь на `@s`, выполняющийся на самом датапаке
- Датапаки не могут выполнять команды на самих себе

## Требования

- Запущенный локально Ollama (`ollama serve`)
- Установленная хотя бы одна модель (`ollama pull llama3.2`)

## Примеры промтов

```bash
# Система квестов
mcpack -p "система квестов с наградами, используй scoreboard для отслеживания прогресса"

mcpack --new -p "система кастомного оружия с уникальными способностями и кулдаунами"

# Мини-игра
mcpack --new -p "мини-игра 'capture the flag' с командами и счётчиком очков"
```
